# 🚀 从这里开始学习 Oxc

在开始之前，确保你已经：

## 🎮 动手实验

### 实验 1: 修改示例代码

打开 `src/skeleton_01_parser_basics.rs`，尝试：

1. 在 `example_1_variable_declaration()` 中打印更详细的信息
2. 实现 `count_functions()` 函数
3. 运行测试：`cargo test --bin 01_parser_basics`

### 实验 2: 解析自己的代码

创建一个 JavaScript 文件，然后用 Parser 解析：

```bash
# 创建测试文件
echo "const myCode = 'test';" > my_test.js

# 使用官方示例解析
cargo run -p oxc_parser --example parser -- my_test.js
```

### 实验 3: 对比不同的代码

尝试解析以下代码，观察 AST 的区别：

```javascript
// 版本 1
const x = 1;

// 版本 2
let x = 1;

// 版本 3
var x = 1;
```

**思考**: 它们的 AST 有什么不同？

## ❓ 常见问题

### Q: 我是 Rust 初学者，能学吗？

A: 可以！这个学习路径从**使用**开始，你会在过程中逐渐熟悉 Rust。遇到不懂的 Rust 语法，可以边学边查。

### Q: 我需要了解编译原理吗？

A: 不需要！我们从实用角度出发。了解编译原理会有帮助，但不是必须的。

### Q: 需要多长时间？

A: 按照每天 1-2 小时的节奏：
- 第一周：基础使用（5-7 天）
- 第二周：核心概念（7-10 天）
- 第三周：深入方向（根据兴趣）

### Q: 卡住了怎么办？

A:
1. 先查看文档中的"常见问题"部分
2. 在代码中搜索相关实现
3. 在学习群/社区提问
4. 提交 Issue 寻求帮助

### Q: 我可以跳过某些部分吗？

A: 可以！学习路径是建议性的。你可以根据自己的兴趣和目标调整。
